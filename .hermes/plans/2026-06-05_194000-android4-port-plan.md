# 安卓 4.4 (API 19) 兼容重构 Plan

## 目标
- minSdk 26 → 19
- 用户设备：4.4 KitKat 真机
- 屏幕：1920×1080 / 160dpi（已 sdp/xdp/ydp 适配，分辨率不是问题）

## 关键约束
- **Jetpack Compose 完全不能跑**（最低 API 21，生产验证到 23+）→ 全删，改 XML + Fragment
- **Media3 1.0+ 最低 API 21** → 降级到 ExoPlayer 2.19.1（最后支持 16+ 的版本）
- **TensorFlow Lite 2.14 最低 21** → 降到 2.8.0（最后支持 19 的版本）
- **Material3 → Material Components 1.11.0**（最低 19）

## 现状盘点
- 22 个 Kotlin 文件，6186 行
- 4 个 Compose Screen（MusicScreen 802 行最大，HomeScreen/MineScreen/SearchScreen）
- 1 个 Compose LyricsView
- 3 个 Compose Dialog/Drawer（PlaylistDrawer/QueueDialog/AddToPlaylistDialog）
- BottomNav + TopBar Compose 组件
- 1 个 CoverColorPredictor（TFLite 推理）
- ViewModel/Repository/Player/Service 业务层不变

## 分批执行

### Batch 1：脚手架（1-2 天）
**目标**：minSdk 19 改完 + Compose 删完 + 一个空 Activity 能装上

步骤：
1. `app/build.gradle.kts`：
   - `minSdk = 19`
   - 删 `buildFeatures { compose = true }` 和 `composeOptions`
   - 删所有 `androidx.compose.*` 依赖
   - 删 `activity-compose`、`lifecycle-viewmodel-compose`、`navigation-compose`
   - 删 `coil-compose`
   - 加 `androidx.appcompat:appcompat:1.6.1`（最低 14）
   - 加 `com.google.android.material:material:1.11.0`（最低 19）
   - 加 `androidx.constraintlayout:constraintlayout:2.1.4`（最低 14）
   - 加 `androidx.lifecycle:lifecycle-viewmodel-ktx:2.6.2`
   - 加 `androidx.lifecycle:lifecycle-livedata-ktx:2.6.2`
   - 加 `androidx.fragment:fragment-ktx:1.6.2`
   - 加 `androidx.navigation:navigation-fragment-ktx:2.7.5`
   - 加 `io.coil-kt:coil:2.5.0`
   - 替换 `media3-*` → `com.google.android.exoplayer:exoplayer:2.19.1`
   - 替换 `tensorflow-lite 2.14.0` → `tensorflow-lite 2.8.0`
2. `AndroidManifest.xml`：
   - `application` 改 `android:theme="@style/Theme.AppCompat.DayNight.NoActionBar"` 或保留自定义 Material 主题
   - `MainActivity` 改 extends AppCompatActivity
3. `MainActivity.kt`：
   - ComponentActivity → AppCompatActivity
   - 删所有 Compose import
   - `setContent { MainScreen(...) }` → `setContentView(R.layout.activity_main)`
4. 新建 `res/layout/activity_main.xml`：
   - 单 FragmentContainerView 占满
   - BottomNavigationView 在底部
5. 新建 `res/menu/bottom_nav_menu.xml`（3 tab: 推荐/音乐/我的）
6. 新建 `res/values/themes.xml`、`colors.xml`、`strings.xml`
7. 新建 `res/drawable/` 基础 vector drawable（搜索/播放/暂停/上一首/下一首/收藏/列表）
8. 把 MainScreen 临时换成 `MusicFragment` 占位（空 XML + 空 Fragment，能 setText "MinSDK 19 OK"）
9. `gradlew assembleRelease` 验证

**验证**：`./gradlew assembleRelease` 通过，APK 能装上 4.4 真机并显示占位

## 风险点
1. **Media3 → ExoPlayer 2.19.1**：MediaSession API 不同，可能要重写大量 Player/Service 交互代码
2. **TFLite 2.8.0**：API 跟 2.14 有差异，CoverColorPredictor 改写
3. **Compose 状态管理 → LiveData/StateFlow**：collectAsState 替换为 observe，需要小心 lifecycle
4. **Coil compose → Coil**：ImageView 加载方式变化
5. **导航**：Navigation Compose → Navigation Fragment 写法差异
6. **没 4.4 真机回归测试**：本地能编译但实机可能崩（API 19 的怪 bug 多）

## 用户决策点
- **icon 资源**：是用 material-icons-extended（资源多但 APK 增大）还是用核心几个 vector drawable（APK 小但工作量大）
- **封面色预测**：在 19 上保留还是降级为纯图片采样（CPU 推理慢）
- **横屏 1920×1080 适配**：保留还是只做竖屏
